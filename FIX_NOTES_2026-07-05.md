# Fix Notes — 2026-07-05 upload

## Root cause of the crash (from Logs/LogReport_05-07-2026.txt)

Two Windows-environment issues were crashing every request, before your own code
ever ran:

1. **EventLog logging provider — Access Denied.** ASP.NET Core auto-registers a
   Windows Event Log logger at Warning level. Your Windows account can't write to
   the Event Log, so the very first warning-level log message (from HTTPS
   redirection) threw an `AggregateException` and killed the request.
   - **Fix (`Program.cs`):** `builder.Logging.ClearProviders(); builder.Logging.AddConsole(); builder.Logging.AddDebug();`
     right after `WebApplication.CreateBuilder(args)`.

2. **DataProtection keys — Access Denied.** Session cookies need an encryption key
   ring; it was trying to write to `AppData\Local\ASP.NET\DataProtection-Keys`
   (a OneDrive-synced path in your case), which failed.
   - **Fix (`Program.cs`):** key storage now points at `DataProtection-Keys/` under
     the project's content root (durable — not wiped by `dotnet clean` like the
     previous `obj/data-protection-keys` location) instead of the AppData path.

Both fixes are additive lines in `Program.cs` only. Nothing else was touched to fix these two.
